# Summary
`RejectExportRecord` is a plain‑old‑Java‑Object (POJO) that models a single row of the daily or monthly reject reports generated by the Move‑Mediation revenue‑assurance pipeline. It captures file‑level identifiers, customer details, usage context, rejected CDR count, rejected usage amount, and the rejection reason. The object is used by export modules to serialize report data to CSV or other downstream sinks.

# Key Components
- **Class `RejectExportRecord`**
  - Private fields: `callDate`, `callMonth`, `customerNum`, `customerName`, `usageFor`, `propositionOrAddon`, `callType`, `rejectedCDR`, `rejectedUsage`, `reason`.
  - Public JavaBean getters/setters for each field.
  - No business logic; serves solely as a data carrier.

# Data Flow
| Stage | Input | Processing | Output | Side Effects |
|-------|-------|------------|--------|--------------|
| Report Generation | Mediation engine produces reject metrics per file/customer | Mapper populates a `RejectExportRecord` instance per row | List/collection of `RejectExportRecord` objects passed to CSV writer or database loader | None (object is immutable after population) |
| Export | `RejectExportRecord` objects | `getLine()`‑style utility (outside this class) concatenates fields into CSV line | CSV file, S3 bucket, or downstream API payload | I/O to file system or network storage |

# Integrations
- **Mediation Pipeline**: Consumes reject counters from CDR processing modules and creates `RejectExportRecord` instances.
- **Export Service (`RAReportsExport`)**: Iterates over collections of `RejectExportRecord` to write CSV files for downstream consumption (e.g., billing reconciliation, audit).
- **Monitoring/Logging**: May log object state via `toString()` (inherited) for traceability.
- **No direct DB/Queue interaction**: The POJO itself does not perform I/O; integration points reside in surrounding services.

# Operational Risks
- **Missing/Null Fields**: Absence of mandatory fields (e.g., `callDate`) can produce malformed CSV rows. *Mitigation*: Validate objects before serialization.
- **Data Type Mismatch**: `rejectedUsage` is a `float`; precision loss may affect usage calculations. *Mitigation*: Use `BigDecimal` if exactness required.
- **Schema Drift**: Adding/removing columns in downstream reports without updating this POJO leads to alignment errors. *Mitigation*: Version the DTO and enforce schema contracts in CI.

# Usage
```java
// Example creation
RejectExportRecord rec = new RejectExportRecord();
rec.setCallDate("2025-12-07");
rec.setCallMonth("202512");
rec.setCustomerNum("C12345");
rec.setCustomerName("Acme Corp");
rec.setUsageFor("VOICE");
rec.setPropositionOrAddon("BASE");
rec.setCallType("OUTGOING");
rec.setRejectedCDR(12L);
rec.setRejectedUsage(34.5f);
rec.setReason("Invalid IMEI");

// Serialize (performed by external utility)
String csvLine = String.join(",",
    rec.getCallDate(),
    rec.getCallMonth(),
    rec.getCustomerNum(),
    rec.getCustomerName(),
    rec.getUsageFor(),
    rec.getPropositionOrAddon(),
    rec.getCallType(),
    String.valueOf(rec.getRejectedCDR()),
    String.valueOf(rec.getRejectedUsage()),
    rec.getReason()
);
System.out.println(csvLine);
```
