# Summary
`pom.properties` is a Maven‑generated descriptor packaged with the `mnaas` artifact. It provides immutable Maven coordinates (groupId, artifactId, version) and Eclipse‑specific project metadata (projectLocation, projectName). Build tools, IDEs, and runtime classpath resolvers use this file to verify the exact source checkout and version of the MOVE‑Mediation Ingest (MNAAS) component deployed in production.

# Key Components
- **groupId** – Maven group identifier (`com.tcl.move`).
- **artifactId** – Maven artifact identifier (`mnaas`).
- **version** – Artifact version (`1.0`).
- **m2e.projectLocation** – Absolute filesystem path to the Eclipse project source.
- **m2e.projectName** – Eclipse project name (`mnaas`).

# Data Flow
- **Input:** Generated by Maven during the `package` phase; consumes project POM metadata.
- **Output:** Embedded in the JAR/WAR under `META-INF/maven/com.tcl.move/mnaas/pom.properties`.
- **Side Effects:** Enables downstream tools (e.g., Ant build scripts, CI pipelines, runtime classpath scanners) to resolve the exact artifact version and source location.
- **External Services:** None; purely file‑based.

# Integrations
- **Ant build scripts** (`build.xml`) read the Maven coordinates to construct compile‑time and runtime classpaths for development (`mnaas_dev.properties`) and production (`mnaas_prod.properties`) pipelines.
- **IDE (Eclipse m2e)** uses `m2e.projectLocation`/`m2e.projectName` for project linking and incremental builds.
- **CI/CD pipelines** may parse the file to enforce version consistency across environments.

# Operational Risks
- **Stale metadata:** If the file is not regenerated after a version bump, deployments may reference an incorrect artifact version. *Mitigation:* Enforce clean rebuilds (`mvn clean package`) in CI.
- **Hard‑coded absolute path:** `m2e.projectLocation` points to a developer’s local workspace, which is irrelevant in production and may cause confusion. *Mitigation:* Strip or ignore this entry in production packaging profiles.

# Usage
```bash
# Verify artifact metadata
unzip -p mnaas-1.0.jar META-INF/maven/com.tcl.move/mnaas/pom.properties

# Example parsing in a shell script
GROUP_ID=$(grep '^groupId=' pom.properties | cut -d= -f2)
ARTIFACT_ID=$(grep '^artifactId=' pom.properties | cut -d= -f2)
VERSION=$(grep '^version=' pom.properties | cut -d= -f2)
echo "Deploying $GROUP_ID:$ARTIFACT_ID:$VERSION"
```

# Configuration
- No runtime configuration; values are generated from the project's `pom.xml`.
- Optional: Maven profile can exclude `m2e.*` entries for production builds.
